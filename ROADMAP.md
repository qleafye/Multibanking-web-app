# Roadmap проекта StudFi

Этот документ описывает текущее состояние проекта (MVP) и намечает возможные пути его дальнейшего развития.

## Что сделано (MVP на 05.11.2025)

На данный момент реализована основная архитектура и функционал проекта в виде двух независимых микросервисов.

### 1. Backend (Java-сервис)
- **Задача:** Агрегация данных из разных источников.
- **Реализация:**
    - Создан Spring Boot сервис, который эмулирует сбор транзакций из трех разных банков (`ABank`, `SBank`, `VBank`).
    - Предоставляет единый, унифицированный API-эндпоинт `/api/v1/transactions` для получения всех транзакций в простом JSON-формате.
    - Полностью независим от логики анализа.

### 2. Analyst Service (Python-сервис)
- **Задача:** Анализ транзакций и поиск подписок.
- **Реализация:**
    - Создан современный, высокопроизводительный микросервис на **FastAPI**.
    - Реализована четкая структура проекта с разделением логики (`logic.py`), моделей данных (`models.py`) и веб-части (`main.py`).
    - Анализ подписок производится на основе гибкой **"Базы знаний"** (`knowledge_base.json`), которую легко расширять.
    - Реализовано два режима работы:
        - **Free:** Находит подписки.
        - **Pro:** Дополнительно обогащает ответ данными о **лайфхаках по экономии** и **альтернативных подписках**.
    - Автоматически генерируется интерактивная документация API (Swagger UI).

### 3. Интеграция и тестирование
- **Задача:** Проверка совместной работы двух сервисов.
- **Реализация:**
    - Создан скрипт `test_integration.py`, который полностью эмулирует работу фронтенда: запрашивает данные у Java-сервиса и передает их на анализ в Python-сервис.
    - Это позволяет тестировать и демонстрировать всю систему целиком без необходимости в готовом UI.

## Что дальше: План развития

### Ближайшие шаги (Short-term)

1.  **Расширение "Базы знаний" (`knowledge_base.json`):**
    - Добавить 10-15 новых популярных сервисов (стриминг, облачные хранилища, игровые сервисы).
    - Для каждого сервиса найти и добавить реальные лайфхаки (семейные, годовые подписки) и альтернативы.

2.  **Добавление нового клиента банка (Bank D):**
    - **(Ключевой пункт для демонстрации)** Создать в Java-сервисе эмуляцию еще одного, четвертого банка. Это наглядно покажет, как легко масштабируется система: аналитический сервис даже не заметит изменений, а данных для анализа станет больше.

3.  **Разработка Frontend-компонента:**
    - Создать минималистичный UI (например, на React или Vue.js), который будет выполнять роль `test_integration.py`: делать запросы к бэкенду и красиво отображать результаты анализа.

### Среднесрочные шаги (Mid-term)

1.  **Улучшение алгоритма анализа (`logic.py`):**
    - Внедрить нечеткий поиск для названий подписок (например, `thefuzz`), чтобы находить опечатки.
    - Анализировать периодичность и стабильность сумм платежей как дополнительный признак подписки.
    - Добавить категоризацию подписок ("Развлечения", "Работа", "Образование").

2.  **Написание тестов:**
    - Покрыть тестами (например, `pytest`) ключевые функции в `logic.py` для обеспечения стабильности при дальнейших изменениях.

### Долгосрочные шаги (Long-term)

1.  **Реализация интерактивного опроса (Pro-версия):**
    - Создать новый эндпоинт `POST /calculate-savings`, который будет принимать выбор пользователя (какие лайфхаки и альтернативы он хочет применить).
    - Реализовать логику расчета финальной экономии на основе этого выбора.

2.  **Контейнеризация и развертывание:**
    - Использовать `Dockerfile` для упаковки сервисов и развернуть их в облаке (например, с помощью Docker Compose или Kubernetes).

3.  **Создание системы уведомлений:**
    - Добавить функционал, который будет периодически анализировать транзакции в фоне и присылать пользователю push-уведомления о новых найденных подписках или возможностях для экономии.
